## CC6.1 — 逻辑与物理访问控制

| # | 控制项 | 系统实现 | 状态 |
|---|--------|---------|------|
| 1.1 | 唯一用户标识 | RBAC Manager (`src/enterprise/rbac-manager.js`) — 每个用户有唯一 `userId`，支持 6 种预定义角色 | ✅ 已实施 |
| 1.2 | 强密码策略 | PBKDF2-SHA512 (600,000 迭代, OWASP 2025 推荐)，无硬编码默认密码，未配置时生成 20 字符随机密码 | ✅ 已实施 |
| 1.3 | 多因素认证 | **未实现 MFA/TOTP** — 当前仅支持密码 + API Key 双模式，无第二因素 | ❌ 未开始 |
| 1.4 | 会话超时管理 | JWT 1h 过期 + 每用户最大 5 并发会话 + 定时清理过期会话（每分钟） | ✅ 已实施 |
| 1.5 | IP 绑定验证 | JWT Payload 包含 IP，validateToken 时执行 IP 不匹配拒绝 | ✅ 已实施 |
| 1.6 | 暴力破解防护 | 审计日志异常检测：5 次/5分钟失败登录触发 `brute_force` 告警 | ✅ 已实施 |

## CC6.2 — 用户访问授权

| # | 控制项 | 系统实现 | 状态 |
|---|--------|---------|------|
| 2.1 | 基于角色的访问控制 (RBAC) | 完整 RBAC：6 预定义角色 + 自定义角色 + 角色层级继承 + 32 种细粒度权限 | ✅ 已实施 |
