### 定价策略原则

1. **Community 版要足够好用** — 让开发者先爱上产品，再自然转化到 Pro
2. **Pro 版定价要有"明显价值感"** — $99/月 vs 企业自建安全层的人力成本（至少 $5,000/月）
3. **Enterprise 版不公开定价** — 按需定制，每个客户单独报价
4. **提供 30 天 Pro 免费试用** — 降低试用门槛

### 价格锚点对比

| 方案 | 月成本 | 获得的价值 |
|------|--------|-----------|
| 自建安全 Agent（2人月） | ~$20,000 一次性 | 基础 Agent + 基础安全 |
| LangChain + 自建安全层 | ~$8,000/月 (1 FTE) | Agent 框架 + 需持续维护 |
| OpenAI Assistants API | ~$500-5,000/月 | 托管 Agent + 数据上云 |
| **TriCore Pro** | **$99/月** | 完整 Agent + 企业安全 |
| **TriCore Enterprise** | **$2,000+/月** | 全套 + 专属支持 |

## 六、风险与应对

| 风险 | 概率 | 影响 | 应对策略 |
|------|------|------|---------|
| **市场不买账** — 开发者不认为三核架构有价值 | 中 | 高 | 用对比评测和案例证明差异化价值；快速迭代 |
| **大厂入场** — OpenAI/Google 推出更强 Agent 产品 | 高 | 中 | 聚焦"自托管+安全"这个云厂商不做的市场 |
| **开源社区不活跃** | 中 | 中 | 投入社区运营、提供贡献者激励、降低贡献门槛 |
| **企业销售周期过长** | 高 | 中 | 先用 Pro 版（自助订阅）跑通模式，再推 Enterprise |
| **安全漏洞** | 低 | 高 | 安全审计 + Bug Bounty 计划 + 快速修复流程 |
| **Python 生态压制** — Node.js 不被 AI 社区接受 | 中 | 中 | 强调 REST API 语言无关；考虑未来提供 Python SDK |

## 七、长期增长路径

### 从 Wedge 到 Platform

```
Phase 1 (当前): Wedge
  └── "企业级自主 AI Agent 引擎"
      目标: 前 100 个付费客户

Phase 2: 扩展
  └── 插件生态 + 技能市场
      目标: 1000+ 社区插件，形成网络效应
```
