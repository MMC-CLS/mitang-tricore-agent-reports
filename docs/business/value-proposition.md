# TriCore Agent 价值主张 (Value Proposition)

> **产品**: 蜜糖 TriCore Agent — 三核融合智能体  
> **版本**: v1.0.0  
> **目标**: 为 AI 开发者和企业团队提供可自主思考、任务闭环、持续进化的智能体系统

## 二、核心价值

### 一句话价值主张

> **TriCore Agent 是唯一将"自主思考、任务执行、持续进化"三核分离的 AI 智能体系统 — 它不只是回答问题，而是像一个真正的团队成员一样持续运行、积累经验、自我优化。**

### 三大核心价值

#### 1. 自主思考 (Consciousness) — "不只是响应，而是主动觉察"

- **TICK 驱动**: 5 种模式（10s 觉醒 / 30s 任务活跃 / 5min 意识 / 10min 进化 / 20min 空闲），持续感知环境变化
- **焦点栈注意力**: 语义级话题切换，最大深度 4，自动追踪对话上下文
- **L1/L2 双层思考**: 简单问答快速响应 + 复杂问题深度分析，智能分流
- **空闲觉察**: 在无用户输入时自主发现值得关注的事项

> 竞品对比: LangChain 和 AutoGPT 是被动响应的，需要用户每次触发。TriCore 是持续运行的"数字同事"。

#### 2. 任务闭环 (Execution) — "不只是生成文本，而是完成工作"

- **5 阶段任务闭环**: Pending → Planning → Executing → Verifying → Completed/Failed
- **安全沙箱**: 所有操作限制在可控范围内，防止 Agent "越权"
- **并行工具调用 + 自动重试**: 最多 3 次指数退避重试
- **插件生态**: 5 种插件类型（tool/provider/middleware/ui/skill_pack），标准化协议

> 竞品对比: OpenAI Assistants 可以调用工具但缺乏完整的任务生命周期管理。TriCore 提供端到端的任务闭环。

#### 3. 持续进化 (Evolution) — "不只是重复执行，而是越用越聪明"

- **技能沉淀引擎**: LLM 驱动从执行轨迹中提取可复用技能
- **SKILL.md 开放标准**: 可搜索、可分享、可移植的技能格式
- **技能审计**: 自动沉淀的技能默认 pending，必须审计才能激活（安全约束）
- **五层记忆模型**: Hot → Warm → Cold → Exec → Skill，全生命周期管理

> 竞品对比: CrewAI 和 AutoGen 的多 Agent 协作后无知识沉淀。TriCore 每次执行都在积累可复用技能。

## 三、关键差异化

### 与主流方案的差异

| 维度 | TriCore Agent | LangChain | AutoGPT | CrewAI | OpenAI Assistants |
|------|-------------|-----------|---------|--------|-------------------|
| **架构模式** | 三核分离 + 治理层 | 链式框架 | 单 Agent 循环 | 多 Agent 角色扮演 | 托管 Agent |
| **自主运行** | ✅ TICK 持续驱动 | ❌ 被动触发 | ⚠️ 有限循环 | ❌ 被动触发 | ❌ 被动触发 |
| **技能沉淀** | ✅ 自动提取+审计 | ❌ 无 | ❌ 无 | ❌ 无 | ❌ 无 |
| **记忆系统** | ✅ 五层模型+双路召回 | ⚠️ 基础向量存储 | ⚠️ 短期记忆 | ⚠️ 基础 | ⚠️ 托管 |
| **企业安全** | ✅ RBAC+审计+加密 | ⚠️ 需自建 | ❌ 无 | ❌ 无 | ⚠️ 托管 |
| **插件系统** | ✅ 标准化协议+热加载 | ✅ 丰富生态 | ⚠️ 命令式 | ⚠️ 有限 | ✅ Function Calling |
| **部署方式** | ✅ 自托管 Docker | ✅ 自托管 | ✅ 自托管 | ✅ 自托管 | ❌ 仅云 |
| **可观测性** | ✅ Prometheus+Grafana | ⚠️ 需自建 | ⚠️ 基础 | ⚠️ 基础 | ⚠️ 托管 |

### 架构差异化

TriCore 的核心差异在于**三核分离 + 治理层**的架构设计：

```
传统 Agent:  输入 → LLM → 输出（单管道，无状态）
LangChain:   输入 → Chain → Tool → LLM → 输出（有结构，但无自主性）
TriCore:     输入 → 意识核(思考) → 治理层(授权) → 执行核(行动) → 进化核(沉淀)
              ↑_______________________________________________↓
                            持续循环，自我进化
```

这种架构的核心优势：
1. **安全性**: 思考、执行、进化三者分离，任何一核无法越权
2. **可靠性**: 每核职责清晰，故障隔离，不会出现"Agent 幻觉式越权"
3. **可进化性**: 进化核独立运行，不影响意识核和执行核的正常工作

## 四、典型使用场景

### 场景 1: 自动化代码审查与优化

**用户**: 软件开发团队  
**痛点**: Code review 耗时、风格不一致、重复性问题反复出现  

**TriCore 方案**:
1. 通过插件系统安装 `code-reviewer` 工具插件
2. 提交 PR → Webhook 触发 → 意识核分析变更范围 → 执行核运行审查规则 → 输出审查报告
3. 进化核从每次审查中沉淀"常见问题模式"技能
4. 3 个月后，系统自动识别 80% 的常见问题，审查效率提升 3x

**ROI**: 节省每人每周 3-5 小时的 Code Review 时间

### 场景 2: 智能运维助手

**用户**: DevOps/SRE 团队  
**痛点**: 告警疲劳、故障排查耗时长、知识散落在不同工具中  

**TriCore 方案**:
1. 接入 Prometheus Alertmanager → 意识核接收告警，L1 快速判断严重程度
2. L2 深度分析：查询记忆中的历史相似事件、搜索技能库中的修复方案
3. 执行核自动执行诊断命令（安全沙箱限制）、生成排查报告
4. 进化核沉淀"故障诊断技能"供下次使用

**ROI**: MTTR（平均修复时间）降低 40-60%

### 场景 3: 企业内部知识助手

**用户**: 全公司员工  
**痛点**: 文档分散、新人上手慢、重复性问题反复问  

**TriCore 方案**:
1. RAG 引擎加载企业文档（支持 PDF/代码库/Wiki）
2. 意识核理解问题意图 → 焦点栈保持多轮对话上下文
3. 记忆系统积累高频问答，越用越精准
4. 进化核自动提取"FAQ 技能"减少重复推理

**ROI**: 新人上手时间减少 50%，内部支持工单减少 30%

### 场景 4: 自动化数据分析 Pipeline

**用户**: 数据分析团队  
**痛点**: 重复性数据清洗和分析任务多、不同数据源整合复杂  

**TriCore 方案**:
1. 用户提交分析目标 → 执行核规划数据获取/清洗/分析步骤
2. 并行调用多个数据源工具 → 自动重试失败步骤
3. 进化核沉淀"数据分析模板"技能，类似分析目标直接复用
4. 子智能体系统支持多分析任务并行执行

**ROI**: 重复性分析任务自动化率 70%+

### 场景 5: AI 应用的 Agent 后端

**用户**: AI 应用开发者  
**痛点**: 自建 Agent 系统工作量大、安全和可靠性难以保证  

**TriCore 方案**:
1. 作为后端 Agent 引擎，通过 REST API + WebSocket 集成到应用中
2. 开箱即用的 RBAC 权限控制，支持多租户
3. 插件系统扩展自定义工具
4. Prometheus 监控指标直接接入现有运维体系

**ROI**: Agent 功能开发周期从数月缩短到数周

## 六、为什么选择 TriCore？

1. **不只是框架，是持续运行的智能体** — LangChain 是工具包，TriCore 是"数字员工"
2. **三核分离，安全可控** — 思考、执行、进化各司其职，企业可审计每一步
3. **越用越聪明** — 技能沉淀和记忆系统让系统持续进化，而非静态工具
4. **开箱即用** — Docker 一行命令启动，5 分钟体验完整功能
5. **企业就绪** — RBAC + 审计 + 加密 + 沙箱，安全合规不妥协
6. **开放生态** — MIT 开源、标准化插件协议、SKILL.md 开放格式
